## 飞书文件分享链接限制

用户尝试分享飞书文件链接（`/file/xxx` 格式），发现飞书 API 无法直接读取这种格式的分享链接。

**原因**：`/file/xxx` 是飞书的文件分享链接，需要登录/权限才能访问。飞书 API 只支持：
- `/docx/xxx` 格式的文档链接
- `/wiki/xxx` 格式的知识库链接

**解决方案**：创建一个带安全验证的文件上传页面，让用户直接上传文件。

## 文件上传服务

创建了带 token + 时间戳验证的文件上传服务：

**组件**：
- `upload_server.js`：Node.js 上传服务，监听 5001 端口
- Docker 镜像：`upload-server`（基于 node:22-bookworm-slim）
- Nginx 代理：通过 nginx-8888 的 `/upload/` 路径代理到后端

**安全机制**：
- Token + 时间戳验证（sha256）
- 链接默认 1 小时过期
- 单文件最大 50MB
- 支持常见文件格式

**文件位置**：
- 服务代码：`~/.openclaw/workspace/webroot/upload_server.js`
- Dockerfile：`~/.openclaw/workspace/webroot/Dockerfile.upload`
- Nginx 配置：`~/.openclaw/workspace/webroot/nginx-upload.conf`

**生成上传链接命令**：
```bash
TIMESTAMP=$(($(date +%s) + 3600))
TOKEN=$(echo -n "openclaw-upload-secret-2024:${TIMESTAMP}" | sha256sum | cut -c1-32)
echo "http://<公网IP>:8888/upload/${TOKEN}/${TIMESTAMP}"
```

**上传文件保存位置**：`/tmp/uploads/`

**遇到的 Bug**：Node.js 中 `generateToken` 函数在拼接时多了个冒号（`SECRET:timestamp:`），导致与 shell 生成的 token 不匹配。已修复。
