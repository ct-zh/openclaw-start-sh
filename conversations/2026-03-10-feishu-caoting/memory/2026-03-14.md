# 2026-03-14 Memory

## 钉钉 Stream 连接 Bug 诊断

### 问题现象
- `/var/log/messages` 每秒写入约 130 条日志
- 日志内容：`TERMINATE SOCKET: Ping Pong does not transfer heartbeat within heartbeat intervall`
- 10 分钟内产生 542 次重连

### 根本原因
`dingtalk-stream` 库 (v2.1.4) 的 bug：

1. 旧连接心跳超时后调用 `socket.terminate()` 但**没有清理心跳定时器**
2. 重连后，旧定时器错误地 terminate 了新 socket
3. 导致连接成功后仅 ~20ms 就被断开，然后无限循环

```javascript
// 问题代码位置
// ~/.openclaw/extensions/dingtalk/node_modules/dingtalk-stream/dist/client.cjs
// 第 147 行
if (this.isAlive === false) {
  console.error("TERMINATE SOCKET...");
  return this.socket.terminate();  // 没有清理 heartbeatIntervallId
}
```

### 修复方案
修改 `~/.openclaw/extensions/dingtalk/src/channel.ts` 第 1482 行：

```javascript
const client = new DWClient({
  clientId: config.clientId,
  clientSecret: config.clientSecret,
  debug: config.debug || false,
  keepAlive: false,  // 改为 false，禁用有 bug 的心跳检测
});
```

钉钉 Stream 服务器会发送 `KEEPALIVE` 消息，禁用客户端心跳不影响连接稳定性。

### 相关文件
- 配置：`~/.openclaw/openclaw.json`
- 插件源码：`~/.openclaw/extensions/dingtalk/src/`
- 库代码：`~/.openclaw/extensions/dingtalk/node_modules/dingtalk-stream/dist/client.cjs`

### 日志空间清理命令
```bash
sudo journalctl --vacuum-size=500M
sudo truncate -s 0 /var/log/messages
```
