## Gateway Token Mismatch 问题排查

**现象**：Gateway 连接失败，报错 `unauthorized: gateway token mismatch`

**根因**：systemd user service 文件中硬编码了环境变量 `OPENCLAW_GATEWAY_TOKEN=test`，覆盖了配置文件中的 token。

- 服务文件位置：`~/.config/systemd/user/openclaw-gateway.service`
- 环境变量值：`test`
- 配置文件值：`45efcb45c5776dc45cf187d72550dbdd`

**解决方案**：修改 service 文件中的 `OPENCLAW_GATEWAY_TOKEN` 为配置文件中的值，然后执行 `systemctl --user daemon-reload && systemctl --user restart openclaw-gateway`。

**后续问题**：Token 修复后，出现 "pairing required" 错误。由于 CLI 需要 gateway 连接才能批准配对，但 gateway 需要配对才能连接，形成循环依赖。

**配对循环解决方案**：手动修改 `~/.openclaw/devices/paired.json`，添加设备配对记录：

```json
{
  "deviceId": "...",
  "publicKey": "...",
  "token": "<gateway.auth.token>",
  "role": "operator",
  "scopes": ["operator.admin", "operator.approvals", "operator.pairing"],
  ...
}
```

**排查过程**：
1. 配置文件中 `gateway.auth.token` 和 `gateway.remote.token` 一致
2. Gateway 重启后问题依旧
3. 通过 `env | grep -i token` 发现环境变量 `OPENCLAW_GATEWAY_TOKEN=test`
4. 通过 `systemctl --user cat openclaw-gateway` 定位到 service 文件
5. 配对循环通过手动修改 paired.json 解决

**环境变量优先级**：`OPENCLAW_GATEWAY_TOKEN` 环境变量 > 配置文件 `gateway.auth.token`

**设备配对文件位置**：`~/.openclaw/devices/paired.json` 和 `~/.openclaw/devices/pending.json`
